## Как работает аналитика


### Валидация
Берёт каждый вопрос из загруженного экзамена, сверяет со связанной темой силлабуса. Смотрит на средний балл, количество попыток и пересечение ключевых слов. Если балл ниже 30% или выше 95% от максимума — ставит предупреждение. Если вопрос вообще не привязан к силлабусу — тоже флаг.

### Анализ качества
Считает индекс сложности для каждого вопроса: `avg_score / max_score`. Ниже 0.3 — слишком сложный (`too_hard`), выше 0.8 — слишком лёгкий (`too_easy`). Показывает распределение по всему экзамену.

### Корреляции
Считает item-total correlation (Pearson r) между баллом за вопрос и суммарным баллом студента за весь экзамен (с вычетом самого вопроса). Если `|r| < 0.12` — вопрос не различает сильных и слабых студентов, флаг `low_corr`.

### Студенты
Смотрит на паттерны ответов каждого студента. Если провалы по всем вопросам одной темы — `systemic` (системная проблема с материалом). Если провалы только на отдельных вопросах — `spot` (локальный сбой). Норма — `ok`.

### Семантика
Сравнивает текст вопроса с названием и ключевыми словами темы силлабуса через TF-IDF + векторные embeddings (Python FastAPI). Выдаёт уровень совпадения: `high` / `medium` / `low`. Требует запущенного embeddings API.

### Анализ критериев
Берёт загруженные критерии оценки, смотрит на средние баллы студентов по каждому критерию. Уровни: `excellent` (≥90%), `good` (≥75%), `satisfactory` (≥60%), `poor` (<60%).

### Ответы студентов
Анализирует текстовые ответы студентов: определяет язык ответа, считает процент плагиата между ответами разных студентов на один вопрос. `plagiarism_rate > 10%` — подозрение на списывание.

**Требования:** XAMPP (PHP 8+, MySQL), Node.js, Python 3.10+
